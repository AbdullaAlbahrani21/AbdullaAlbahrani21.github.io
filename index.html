Lem-in
OVERVIEW

A digital ant farm simulation in Go. Given a colony map of rooms and tunnels, the program finds the optimal set of paths to move N ants from a start room to an end room in the fewest possible turns — avoiding collisions and traffic jams along the way. The challenge is both algorithmic (graph pathfinding) and logistical (multi-agent flow optimization).

KEY FEATURES
  • Parses a custom colony file format defining rooms, tunnels, and ant count
  • Finds multiple non-conflicting paths simultaneously using BFS-based algorithms
  • Schedules ant movement to minimize total turns — not just shortest single path
  • Each room holds at most one ant per turn (except start and end)
  • Handles all invalid input cases: duplicate rooms, unknown links, missing start/end, self-links
  • Outputs every ant's movement per turn in a clean, structured format

Net-Cat
OVERVIEW

A Go reimplementation of the classic NetCat utility, built as a multi-client TCP group chat server. The server handles up to 10 simultaneous connections, broadcasts messages to all connected clients, and maintains a full message history that new joiners receive on connection. All concurrency is handled with goroutines and mutexes.

KEY FEATURES
  • TCP server that accepts up to 10 concurrent client connections
  • Clients choose a username on join — empty names are rejected
  • All messages timestamped in the format [YYYY-MM-DD HH:MM:SS][name]
  • New clients receive the full previous message history on connect
  • Server broadcasts join and leave notifications to all active clients
  • Empty messages from clients are not broadcast
  • Goroutines + mutexes ensure safe concurrent access to shared state
